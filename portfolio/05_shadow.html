Project Shadow Mapping

3D Graphics

Project Overview

This project is an interactive 3D graphics demonstration focused on implementing real-time Shadow Mapping. The primary objective was to simulate realistic, dynamic shadows cast by a perspective light source. By utilizing a two-pass rendering pipeline, the scene is first rendered from the light's point of view to generate a depth map, which is then used in the second pass to determine if a fragment is in shadow. The project also features real-time parameter tuning via ImGui and is fully compatible with both Windows x64 and WebGL environments.

Tasks Completed

  • Camera & View Matrices: Implemented highly optimized ViewMatrix and ToWorldMatrix calculations using dot products to transform points across camera and world spaces.
  • Texture Enhancements: Expanded the Texture class to support depth map generation (LoadAsDepthTexture) with specific comparison parameters (GL_COMPARE_REF_TO_TEXTURE) for hardware-accelerated shadow lookups.
  • FrameBuffer Management: Developed a custom FrameBuffer class to handle off-screen rendering for the shadow pass, accommodating both depth and RGBA8 color formats.
  • Two-Pass Shadow Logic: Constructed the core C++ rendering logic to record scene depth from the light's perspective (Pass 1) and apply shadows during the main camera render (Pass 2).
  • GLSL Shaders: Wrote custom vertex and fragment shaders to compute shadow coordinates, perform projection texture lookups (textureProj), and apply advanced visual effects like Gamma Correction and Exponential Fog.
  • Debugging Visualizers: Implemented visual debugging tools to render the light's viewing frustum in 3D space and display the generated 2D depth texture directly on the screen.
  • Cross-Platform Fixes: Resolved strict type-casting issues (signed vs. unsigned integers) for robust WebGL compilation using Emscripten.

Reflection

Implementing shadow mapping was a challenging but incredibly rewarding experience. One of the primary difficulties was ensuring the accurate transformation of coordinates between World, View, and Shadow spaces. Relying on fundamental math (like dot products) instead of pre-built inverse functions for the Camera class greatly improved my understanding of spatial transformations.

Additionally, transitioning the project from a Windows environment to a WebGL build introduced unexpected compiler strictness, such as implicit conversion warnings with Emscripten. Addressing these issues not only made the codebase more robust but also taught me the importance of writing safe, cross-platform C++ code. Ultimately, this project solidified my grasp of multi-pass rendering, framebuffers, and the mathematical foundations of real-time lighting.
