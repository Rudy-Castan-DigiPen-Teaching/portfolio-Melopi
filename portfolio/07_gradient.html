Project Gradient Noise

3D Graphics

Project Overview

In this project, I implemented an Improved 3D Gradient (Perlin) Noise algorithm in GLSL to procedurally generate seamless textures and dynamic 3D surfaces. The primary objective was to extend a custom C++ graphics framework to support advanced texture formats (e.g., R32F) and offscreen framebuffer rendering. By computing noise values entirely on the GPU and utilizing techniques like quintic smoothing, I successfully generated highly organic, continuous patterns including Fractal Sum, Turbulence, Marble, and Wood. This project significantly deepened my understanding of procedural generation and the seamless integration between OpenGL framework logic and complex shader algorithms.

Tasks Completed

  • Implemented Improved 3D Perlin Gradient Noise in GLSL with 12 fixed gradient directions and quintic fading.
  • Generated procedural seamless textures and diverse noise patterns (Fractal, Turbulence, Marble, Wood).
  • Extended the C++ graphics framework to support new color formats (RGBA32F, R32F) for the Framebuffer and Texture loading systems.
  • Developed vertex and fragment shaders to visualize the 3D noise both as a 2D texture slice and as a displaced 3D terrain surface.

Reflection

One of the most significant challenges was ensuring the seamless tiling of the noise and correctly matching the vertex attribute locations between the C++ framework and the GLSL shaders. Debugging visual artifacts, such as flat gray planes or segmented ridges, taught me the critical importance of proper uniform binding (like tile scaling) and precise interpolation methods (quintic fade) in noise functions. Through resolving these issues, I gained a much stronger grasp of the OpenGL pipeline and shader debugging.
