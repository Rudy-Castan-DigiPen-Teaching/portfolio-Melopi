Project Hello Quad

3D Graphics

Project Overview

This project serves as a foundational exercise in developing a modern graphics rendering pipeline using OpenGL 3.3 and WebGL 2. The primary objective was to set up a robust framework capable of rendering a textured quad and to implement custom, dynamic animations through programmable shaders. This foundation will be used for future graphics programming assignments.

Tasks Completed

  • Vertex Shader Animation: Created a dynamic vertex shader that applies a time-based sine and cosine function to the vertex positions, resulting in a continuous "wobbling" effect. Additionally, I integrated mouse interaction to allow the quad to follow the user's cursor in Normalized Device Coordinates (NDC).
  • Fragment Shader Animation: Developed a fragment shader that utilizes a time variable to smoothly cycle the final output color by interpolating (`mix`) between the custom texture data and the quad's original vertex colors.
  • Cross-Platform Deployment: Successfully configured the CMake build system to compile the C++ codebase into a single HTML executable using Emscripten for web browser deployment.

Reflection

One of the most significant challenges in this project was navigating the transition from a native C++ build environment to a WebAssembly deployment. I encountered several issues related to CMake configuration and missing dependencies (like Ninja and Emscripten toolchains). Resolving these issues deepened my understanding of how cross-compilation works.

Additionally, while writing the shader code, I initially struggled with mapping screen-space mouse coordinates to OpenGL's -1.0 to 1.0 coordinate system. Fixing this bug not only achieved the desired interactive effect but also solidified my practical understanding of rendering pipelines and coordinate spaces.
